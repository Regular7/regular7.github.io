an example

In the example above, we see a Regular7 header file. Header files are a type of a Regular7 block marking a directory and it's subdirectories as a Regular7 interpretable repository.

Everytime when you start a new R7 project, or would like to share your work based on some other technology, you will need to write such a header file. Do not worry, the R7 interpreter will help you out fill most of this stuff interactively.

Specifically, the code above displays some of the interesting features of the language:

  • Procedures - the first entry in the header block is a description of a Regular7 procedure call. Here the procedure system.get is invoked with implicit argument (date) taken from the name of the datapoint.

    Contrast this with the third entry, where the parameter to system.get was passed explicitly as (machine-name) and bound to nom.

  • Data transfer operators - also in the first entry, we can see the funny <= operator. This guy tells the interpreter to recompute the data, in contrast to the = transport operator, which will take the right hand expression, compute it, and store it in the R7 repository.

    By choosing carefully between these two operators, you can share your work with other people and their machines, sending them expensive to compute data along with the instructions to compute it, if need be.

in development

This is a very young project, seeking contributors interested in helping people use computers and share their work with other people. We have many features planned and under cosideration, here are some of them:

  • Hosted interpreter model for embedding (a) R7 conforming interpreter into your app
  • If you don't want to embed an interpreter, we also plan to provide a web API for commanding an interpreter
  • Inference engine for interactive exploration of R7 repositories
  • The human procedure standard library, for teaching computers about humans and their domain
